    Teamplay - 팀 업무 통합 관리 도구¶

    1. 프로젝트 개요¶

    1.1 목적¶

    Dooray 기반 팀 업무 진행사항을 실시간으로 추적하고, 주간/월간 보고서를 자동 취합하는 통합 데스크톱 애플리케이션

    1.2 팀 현황¶

    • 팀 규모: 3명 (정재화, 김보람, 김지인)
    • 특징: 소규모 팀이지만 작업량이 많음
    • 핵심 니즈: 스케줄을 절대 놓치면 안 됨

    1.3 핵심 가치¶

    • 진행사항 정확한 트래킹: 팀원들이 댓글로 작성한 업무 내용이 자동으로 Gantt 차트에 반영
    • 스케줄 놓침 방지: 마감 임박 알림, 미진행 업무 하이라이트, 담당자별 업무 과부하 감지
    • 월별 업무 파악: 프로젝트/담당자별 업무 현황을 한눈에 파악
    • 보고서 자동화: 댓글 취합 → 본문 업데이트 → 주간/월간 보고 문서 생성

    1.3 통합 대상¶

    기존 프로젝트 주요 기능 통합 방식
    Team Schedule Manager Gantt 차트, 다양한 뷰, Excel 연동 기반 (Base)
    DoorayMCP 댓글 파싱, 표 취합, 보고서 생성 기능 병합

    3. 기술 스택¶

    3.1 선택 기술¶

    구분 기술 선택 이유
    런타임 Electron 로컬 파일 접근, CORS 우회, 크로스 플랫폼
    프론트엔드 HTML5 + Vanilla JS Team Schedule Manager 기반 활용
    백엔드 Node.js (Electron Main) 파일 시스템, Dooray API 직접 호출
    API 연동 Dooray MCP + REST API 댓글 조회, 업무 조회/수정
    데이터 저장 JSON 파일 + localStorage 로컬 우선, 향후 서버 연동 대비
    빌드 electron-builder Windows exe 배포

    4. 핵심 기능¶

    4.1 기존 기능 (Team Schedule Manager)¶

    기능 설명 상태
    Gantt 차트 프로젝트별 타임라인 시각화 유지
    캘린더 뷰 월별 업무 조회 유지
    팀 Gantt 팀원별 업무 시간표 유지
    프로젝트 타임라인 프로젝트 진행도 유지
    Task CRUD 업무 생성/수정/삭제 유지
    Work 마크 날짜별 작업 표시 유지
    Excel 연동 임포트/내보내기 유지
    자동 백업 정기 저장 유지

    4.2 새로운 기능 (DoorayMCP 통합)¶

    기능 설명 우선순위
    댓글 동기화 Dooray 댓글 자동 조회 및 파싱 P0
    자동 Task 추가 "새 업무 등록" 감지 시 Task 생성 P0
    Work 마크 자동화 MM.DD 날짜 파싱 → 해당 날짜 마킹 P0
    표 취합 댓글 표 → 본문 마크다운 표 생성 P1
    보고서 뷰 주간/월간 보고서 미리보기 P1
    Dooray 업데이트 취합 결과를 Dooray 본문에 반영 P2
    실시간 알림 새 댓글 감지 시 알림 P3

    4.3 스케줄 놓침 방지 기능 (소규모 팀 특화)¶

    3명이 많은 작업량을 처리하므로, 스케줄을 놓치지 않도록 하는 특화 기능

    기능 설명 우선순위
    마감 임박 알림 D-3, D-1, D-day 시각적 강조 P0
    미진행 업무 하이라이트 시작일 지났는데 Work 마크 없는 업무 빨간 표시 P0
    오늘의 할일 대시보드 오늘 진행해야 할 업무 목록 상단 표시 P0
    담당자별 업무량 경고 특정 기간 업무 과부하 시 경고 P1
    주간 마감 요약 이번 주 마감 업무 알림판 P1
    지연 업무 추적 완료일 지났는데 100% 아닌 업무 별도 표시 P1
    자동 리마인더 앱 시작 시 오늘/내일 마감 업무 팝업 P2

    4.3 기능 상세¶

    4.3.1 댓글 동기화¶

    [입력] Dooray 업무 URL
         │
         ▼
[처리] Dooray MCP API 호출
         │
         ├─ 댓글 목록 조회
         ├─ 프로젝트 링크 추출
         ├─ 표 파싱 (금주/다음주/이슈)
         └─ 날짜 추출 (MM.DD 패턴)
         │
         ▼
[출력] ParsedComment[]
       - projectUrl: dooray://...
       - projectName: "로우바둑이"
       - thisWeek: ["01.27 작업A", "01.28 작업B"]
       - nextWeek: "검토 예정"
       - issues: "없음"
       - teamShare: "새 업무 등록"

    4.3.2 자동 Task 추가¶

    // "팀 공유사항"에 "새 업무 등록" 포함 시
if (comment.teamShare.includes('새 업무 등록')) {
    // dooray:// URL에서 원격 업무 정보 조회
    const remoteTask = await fetchRemoteTaskInfo(comment.projectUrl);

    // Gantt 차트에 새 Task 추가
    TaskManager.createTask({
        name: remoteTask.subject,
        project: remoteTask.project,
        assignees: [comment.author],
        startDate: remoteTask.startDate,
        endDate: remoteTask.endDate,
        doorayUrl: comment.projectUrl
    });
}

    4.3.3 Work 마크 자동화¶

    // 금주 진행내용에서 날짜 추출
// "01.27 리소스 작업" → 01.27에 Work 마크
const datePattern = /(\d{2}\.\d{2})\s+(.+)/g;
let match;
while ((match = datePattern.exec(thisWeekContent)) !== null) {
    const [_, dateStr, workContent] = match;
    const workDate = parseDate(dateStr); // MM.DD → Date 객체

    WorkSync.addWorkMark(taskId, workDate, workContent);
}

    5. 데이터 모델¶

    5.1 Task (업무)¶

    interface Task {
    id: string;
    name: string;                    // 업무명
    project: string;                 // 프로젝트
    category?: string;               // 카테고리
    organization?: string;           // 조직
    platform?: string;               // 플랫폼
    assignees: string[];             // 담당자들
    startDate: Date;                 // 시작일
    endDate: Date;                   // 완료일
    progress: number;                // 진행률 (0-100)
    priority?: string;               // 우선순위

    // Dooray 연동
    doorayUrl?: string;              // Dooray 업무 URL
    doorayProjectId?: string;        // Dooray 프로젝트 ID
    doorayPostId?: string;           // Dooray 업무 ID
    wikiUrl?: string;                // 위키 URL

    // 보고서용
    thisWeek?: string;               // 금주 진행내용
    nextWeek?: string;               // 다음주 주요 액션
    issues?: string;                 // 이슈
}

    5.2 WorkLog (작업 로그)¶

    interface WorkLog {
    taskId: string;
    date: string;                    // YYYY-MM-DD
    content: string;                 // 작업 내용
    source: 'manual' | 'dooray';     // 입력 출처
    syncedAt?: Date;                 // 동기화 시간
}

    5.3 WeeklyReport (주간보고)¶

    interface WeeklyReport {
    weekStart: Date;                 // 주 시작일
    weekEnd: Date;                   // 주 종료일
    doorayUrl: string;               // 원본 Dooray URL

    entries: ReportEntry[];          // 보고 항목들
    generatedAt: Date;               // 생성 시간
    updatedToDooray: boolean;        // Dooray 업데이트 여부
}

interface ReportEntry {
    organization: string;
    taskName: string;
    taskUrl: string;
    thisWeek: string;
    nextWeek: string;
    issues: string;
    authors: string[];
    startDate?: string;
    endDate?: string;
    progress?: number;
}
